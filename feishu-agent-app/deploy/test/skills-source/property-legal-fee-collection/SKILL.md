# 物业费催缴涉法处理 Skill

## 适用场景

业主长期欠缴物业费、多次催缴无果、拟发送催缴函、拟起诉物业费、业主以服务质量问题拒缴、业主要求减免物业费。

## 处理重点

整理欠费事实；识别拒缴理由；列出证据材料；生成催缴话术和催缴函草稿；如拟起诉，输出起诉前检查清单和诉讼时效提醒。

## 输出必须包含

欠费事实整理、争议焦点、证据材料清单、催缴话术、催缴函草稿、起诉前检查清单、诉讼时效提醒、是否建议升级法务。

## 表达边界

不建议威胁式催收；不建议公开披露欠费业主个人信息；起诉、减免、和解、停供服务等事项须按公司制度和法务意见处理。


## 通用边界

- 本 Skill 仅用于物业企业涉法事项的内部辅助：场景识别、事实整理、风险提示、材料清单、函件/报告/台账草稿。
- 不替代法务、律师、管理层或政府主管部门的最终判断。
- 涉及对外发送、责任认定、赔偿承诺、合同解除、项目撤场、行政复议/诉讼、诉讼策略等，必须提示人工审核。
- 对用户输入的截图、聊天记录、函件、合同、政府文书、案件材料，只能基于材料做初步分析；不确定事实必须标注“待核实”。

## 统一风险等级

| 等级 | 含义 | AI 可做 | 升级要求 |
|-|-|-|-|
| L1 低风险 | 普通模板、内部材料整理、一般咨询 | 输出模板和建议 | 不一定升级 |
| L2 中风险 | 一般业主争议、普通合同履约、物业费催缴 | 输出建议、材料清单、函件草稿 | 视情况升级 |
| L3 高风险 | 行政处罚、诉讼仲裁、合同解除、赔偿争议、业委会改聘 | 输出初稿和风险提示 | 必须法务审核 |
| L4 重大风险 | 人身伤亡、重大财产损失、消防/电梯重大事故、媒体曝光、群体事件、政府重点监管 | 只做应急清单、事实整理、升级建议 | 立即升级管理层和法务 |

## 弱模型稳定执行规则（必须遵守）

1. **禁止只给原则**：必须输出可执行清单、材料清单、风险等级和下一步动作。
2. **禁止只列 Skill 名称**：如涉及组合执行，必须逐步给出每个步骤的判断结果和输出。
3. **禁止确定性法律定性**，除非用户明确提供已经法务确认的结论。不得直接说：
   - “构成违法”
   - “构成不正当竞争”
   - “物业无需承担责任”
   - “物业应当赔偿”
   - “合同可以解除”
   - “业主大会决议无效”
   - “对方必须返还”
   - “可以直接强制清理”
4. 替代表述应使用：
   - “存在……风险”
   - “需进一步核查……”
   - “建议由法务评估……”
   - “可作为内部初步判断……”
   - “不宜在未核实前作确定性表述……”
5. **一线优先**：每次输出最后必须有【你下一步只需要做什么】，用 3-7 条短句写清楚。
6. **缺材料也要先给动作**：紧急或高风险事项，不得因为材料不足而只追问；先给立即动作，再列需补充信息。
7. **引用材料要标注来源**：从用户材料中抽取的日期、金额、期限、主体、案号等，必须标注“来源：用户材料/待人工复核”。


## 起诉准备度评分（强制）

```text
【起诉准备度】完整 / 基本完整 / 不完整
- 收费依据：完整/缺失
- 欠费明细：完整/缺失
- 业主身份与房屋信息：完整/缺失
- 催缴记录与送达凭证：完整/缺失
- 服务履约记录：完整/缺失
- 拒缴理由及回应：完整/缺失
```

## 催缴函最小模板

必须包含：欠费主体、房号、欠费期间、欠费金额、收费依据、付款期限、联系方式、保留依法主张权利的表述，并避免威胁式语言。
