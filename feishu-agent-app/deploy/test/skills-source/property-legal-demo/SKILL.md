# 法律摆渡人智能体比赛演示能力

本能力面向演示、培训和试点推广，用来直观看到“智能体在组织物业涉法工作”。日常一线使用仍应从法律摆渡人智能体总控入口进入，而不是让员工手动挑选后台 Skill。

## 支持演示场景

1. 业委会改聘 + 共有收益 + 第三方翘盘
2. 消防整改通知 + 行政监管
3. 业主摔伤/车辆受损 + 突发事件 + 赔偿沟通

## 演示输出结构

```text
【一、演示输入】
给出一段一线自然语言或材料摘要。

【二、总入口识别】
展示 property-legal-dispatcher 如何识别场景和风险等级。

【三、Skill 编排路径】
列出子 Skill 执行顺序。

【四、逐步执行结果】
每一步必须包含：目的、判断、材料清单、输出草稿、风险提示。

【五、最终处置包】
- 一线立即动作
- 材料与证据清单
- 沟通口径/函件草稿
- 内部汇报摘要
- 升级提醒

【六、可展示的系统价值】
- 降低一线使用门槛
- 多 Skill 自动编排
- 标准化输出
- 风险边界清晰
```

## 演示要求

- 不追求法律结论，追求流程完整和风险可控。
- 必须突出“员工只需自然语言描述，系统自动分诊”。
- 必须展示“不是只生成文字，而是输出处置包”。
- 必须展示“L3/L4 自动升级法务/管理层”。
