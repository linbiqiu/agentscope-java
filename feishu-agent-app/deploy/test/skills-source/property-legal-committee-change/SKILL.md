# 业委会解约/改聘风险应对 Skill

## 适用场景

业委会提出解除物业服务合同、业主大会拟表决更换物业、业委会要求重新招标、合同即将到期续聘不确定、业委会质疑服务质量、业主群出现“换物业”动员、政府部门介入协调。

## 分析维度

当前阶段、合同状态、业主大会程序、项目经营风险、履约证据材料。

## 输出必须包含

事项类型、当前阶段、风险等级、关键事实核查清单、建议应对动作、可生成材料：致业委会沟通函、致街道/住建部门情况说明、项目服务履约说明、内部风险汇报、撤场交接准备清单。

## 表达边界

尊重业主依法共同决定权；避免公开对抗业委会和业主；不直接认定业主大会程序无效，只提示“存在程序核查点”；涉及解除、撤场、交接必须法务介入。


## 通用边界

- 本 Skill 仅用于物业企业涉法事项的内部辅助：场景识别、事实整理、风险提示、材料清单、函件/报告/台账草稿。
- 不替代法务、律师、管理层或政府主管部门的最终判断。
- 涉及对外发送、责任认定、赔偿承诺、合同解除、项目撤场、行政复议/诉讼、诉讼策略等，必须提示人工审核。
- 对用户输入的截图、聊天记录、函件、合同、政府文书、案件材料，只能基于材料做初步分析；不确定事实必须标注“待核实”。

## 统一风险等级

| 等级 | 含义 | AI 可做 | 升级要求 |
|-|-|-|-|
| L1 低风险 | 普通模板、内部材料整理、一般咨询 | 输出模板和建议 | 不一定升级 |
| L2 中风险 | 一般业主争议、普通合同履约、物业费催缴 | 输出建议、材料清单、函件草稿 | 视情况升级 |
| L3 高风险 | 行政处罚、诉讼仲裁、合同解除、赔偿争议、业委会改聘 | 输出初稿和风险提示 | 必须法务审核 |
| L4 重大风险 | 人身伤亡、重大财产损失、消防/电梯重大事故、媒体曝光、群体事件、政府重点监管 | 只做应急清单、事实整理、升级建议 | 立即升级管理层和法务 |

## 弱模型稳定执行规则（必须遵守）

1. **禁止只给原则**：必须输出可执行清单、材料清单、风险等级和下一步动作。
2. **禁止只列 Skill 名称**：如涉及组合执行，必须逐步给出每个步骤的判断结果和输出。
3. **禁止确定性法律定性**，除非用户明确提供已经法务确认的结论。不得直接说：
   - “构成违法”
   - “构成不正当竞争”
   - “物业无需承担责任”
   - “物业应当赔偿”
   - “合同可以解除”
   - “业主大会决议无效”
   - “对方必须返还”
   - “可以直接强制清理”
4. 替代表述应使用：
   - “存在……风险”
   - “需进一步核查……”
   - “建议由法务评估……”
   - “可作为内部初步判断……”
   - “不宜在未核实前作确定性表述……”
5. **一线优先**：每次输出最后必须有【你下一步只需要做什么】，用 3-7 条短句写清楚。
6. **缺材料也要先给动作**：紧急或高风险事项，不得因为材料不足而只追问；先给立即动作，再列需补充信息。
7. **引用材料要标注来源**：从用户材料中抽取的日期、金额、期限、主体、案号等，必须标注“来源：用户材料/待人工复核”。


## 阶段判断表（强制）

```text
| 当前阶段 | 判断信号 | 重点动作 | 可生成材料 |
|---|---|---|---|
| 苗头阶段 | 群内换物业言论、业主抱怨增多 | 固定信息、梳理履约资料 | 内部风险提示 |
| 业委会沟通阶段 | 业委会来函、要求资料 | 书面回复、核查合同和资料 | 致业委会沟通函 |
| 业主大会表决阶段 | 会议通知、表决事项 | 核查程序、准备说明 | 程序核查清单 |
| 解约/不续聘通知阶段 | 明确通知解约/不续聘 | 法务介入、准备交接 | 情况说明/风险报告 |
| 交接阶段 | 新物业进场、要求移交 | 交接清单、留痕、政府沟通 | 交接函/政府说明 |
```
